## Notes

- **Email addresses are required** to match exactly - this is how the script maps authors to users
- **Password**: You can set any password you want (users won't need to login for the migration)
- **Other details** (name, country, bio) can be customized - they're just metadata
- **You can use real emails** or test emails - whatever works for your project

## Creating Users in Supabase

### Minimum Setup (1 user):

1. Go to your Supabase dashboard → Authentication → Users
2. Click "Add User" → "Create New User"
3. Enter **any email address** (e.g., `test@example.com`)
4. Set a password (can be anything, like "password123")
5. Click "Create User"

That's it! The script will use this user for all posts.

### Full Setup (11 users for unique authors):

If you want each author to have their own user:
1. Follow the steps above
2. Create additional users with the email addresses listed above (must match exactly)
3. The script will map each post to its corresponding author's user

## What the Script Uses

The script only needs:
- **Email** (for mapping - but will reuse any existing user if not found)
- **User ID** (auto-generated by Supabase)

It does NOT need:
- Bio (not stored in auth.users)
- Country (not stored in auth.users)
- Display name (optional - you can set it in user metadata if you want)

After creating at least 1 user, run the migration script:
```bash
node scripts/migrate-static-data.js
```
